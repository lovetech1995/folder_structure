## 2.0 ENGINEERING STANDARDS

### 2.1 NAMING CONVENTIONS (STRICT)

| Entity            | Case/Format      | Example                                  |
| :---------------- | :--------------- | :--------------------------------------- |
| **Files**         | snake_case       | `form_input.js`, `use_action_default.js` |
| **Components**    | PascalCase       | `BailiffUpload`, `DialogPayment`         |
| **Hooks**         | use + PascalCase | `useActionDefault`, `useJobDetailFlow`   |
| **Functions**     | camelCase        | `handleUpdate`, `createQr`               |
| **Variables**     | camelCase        | `loading`, `userInfo`                    |
| **Constants**     | UPPERCASE        | `FLOWS.DEPOSIT`                          |
| **Redux Type**    | UPPER_SNAKE_CASE | `USER_LOGIN_SUCCESS`                     |
| **Redux Actions** | camelCase        | `bailliff_job.action.js`                 |

### 2.2 CODING STYLE & FORMATTING

- **Indentation**: 2 Spaces. (No tabs).
- **Strings**: Double Quotes (`""`).
- **Semicolons**: Always used (`;`).
- **Structure**: Empty lines between all logical sections.
- **Sections**: **MANDATORY** comment headers for all files (see Templates).

### 2.3 IMPORT ORDER

1.  React / Third-party Libs
2.  Local Components
3.  Redux (Actions / Selectors)
4.  Utils / Hooks / Config

## 3.0 IMPLEMENTATION PATTERNS (TEMPLATES)

### 3.1 COMPONENT STANDARD (`src/component` or `src/screen/.../component`)

**Rules:** UI Only. No heavy logic.

```javascript
import React, { useState, useEffect } from "react";
import { Button, Row, Col } from "antd";
// Local imports
import SomeComponent from "component/some_component";
import useSomeHook from "hook/useSomeHook";
import { someSelector } from "store/some.selector";
import "./style.css";

const MyComponent = (props) => {
  // -------------------------- VAR --------------------------
  const someVar = "value";

  // -------------------------- STATE --------------------------
  const [loading, setLoading] = useState(false);

  // -------------------------- REDUX --------------------------
  const someData = useSelector(someSelector);

  // -------------------------- FUNCTION --------------------------
  const handleSomething = () => {
    // UI-only logic
  };

  // -------------------------- EFFECT --------------------------
  useEffect(() => {
    // UI side effects
  }, []);

  // -------------------------- RENDER --------------------------
  const renderItem = () => {
    return <div>Item</div>;
  };

  // -------------------------- MAIN --------------------------
  return <div>{renderItem()}</div>;
};
export default MyComponent;
```

### 3.2 HOOK STANDARD - ACTION (`src/hook`)

**Rules:** Handles user actions, API calls, Form submissions.

```javascript
import { message } from "antd";
import { useState } from "react";
import { useDispatch } from "react-redux";
import { useSearchParams } from "react-router-dom";
// Local imports (actions, selectors, utils)

const useActionDefault = () => {
  // -------------------------- VAR -----------------------------
  const [_, setSearchParams] = useSearchParams();

  // -------------------------- STATE ---------------------------
  const [loading, setLoading] = useState(false);

  // -------------------------- REDUX ---------------------------
  const dispatch = useDispatch();

  // -------------------------- FUNCTION ------------------------
  const handleOpen = () => {
    setSearchParams({ viewOpenAccount: true });
  };

  const handleClose = (form) => {
    setSearchParams({});
    form?.resetFields();
  };

  const handleSubmit = (form) => {
    // Validation & Dispatch Logic
    setLoading(true);
    // dispatch(action(payload, callback...))
  };

  // -------------------------- EFFECT --------------------------
  // -------------------------- DATA FUNCTION -------------------
  // -------------------------- RENDER --------------------------
  // -------------------------- MAIN ----------------------------
  return { handleOpen, handleClose, handleSubmit, loading };
};
export default useActionDefault;
```

### 3.3 HOOK STANDARD - FLOW (`src/hook`)

**Rules:** Handles state transitions, status logic, enums.

```javascript
import { useEffect, useState } from "react";
import { useSelector } from "react-redux";
// Selectors

export const FLOWS = {
  STEP_1: "step_1",
  STEP_2: "step_2",
};

export const useFlowDefault = () => {
  // -------------------------- VAR -----------------------------
  // -------------------------- STATE ---------------------------
  const [flow, setFlow] = useState(FLOWS.STEP_1);

  // -------------------------- REDUX ---------------------------
  const data = useSelector(someSelector);
  const status = data?.status;

  // -------------------------- FUNCTION ------------------------
  // -------------------------- EFFECT --------------------------
  useEffect(() => {
    if (status === "done") setFlow(FLOWS.STEP_2);
  }, [status]);

  // -------------------------- DATA FUNCTION -------------------
  // -------------------------- RENDER --------------------------
  // -------------------------- MAIN ----------------------------
  return { flow };
};
```

## 4.0 WORKFLOW & PROCESS

### 4.1 ADDING FEATURES

1.  **Locate**: Find closest existing feature.
2.  **Clone**: Copy structure (Folder/Files).
3.  **Renaming**: Minimal rename to match new feature context.
4.  **Logic**: Modify specific logic only.

### 4.2 SCREEN MANAGEMENT

- **Numbering**: `00_Nav`, `01_Login`... (Preserve sort order).
- **Scope**:
  - `routePrivate.js`: Authenticated screens.
  - `routePublish.js`: Public screens.
- **Components**: Must inhabit `component/` subfolder if screen-specific.

### 4.3 STATE (REDUX)

- **Folder**: `src/store/[domain]/[module]/`.
- **Files**:
  - `*.action.js`: Thunks, API Calls.
  - `*.reducer.js`: Pure Switch-Case.
  - `*.selector.js`: Memoized selections.
  - `*.type.js`: Constant definitions.
